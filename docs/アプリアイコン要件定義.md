# アプリアイコン要件定義書

## 1. アプリ概要

- **アプリ名**: 集中モードタイマー（TimerApp）
- **プラットフォーム**: macOS
- **アプリ形式**: メニューバーアプリ（NSStatusItem）
- **主要機能**: 
  - 集中タイマー（1分〜120分）
  - ポモドーロタイマー（25分作業 + 5分休憩）

---

## 2. アイコンサイズ要件

macOSアプリアイコンとして、以下のサイズが必要です：

| サイズ | スケール | 解像度 | 用途 |
|--------|---------|--------|------|
| 16×16 | 1x | 16×16 | メニューバー（標準） |
| 16×16 | 2x | 32×32 | メニューバー（Retina） |
| 32×32 | 1x | 32×32 | メニューバー（標準） |
| 32×32 | 2x | 64×64 | メニューバー（Retina） |
| 128×128 | 1x | 128×128 | Dock（標準） |
| 128×128 | 2x | 256×256 | Dock（Retina） |
| 256×256 | 1x | 256×256 | Finder（標準） |
| 256×256 | 2x | 512×512 | Finder（Retina） |
| 512×512 | 1x | 512×512 | システム設定（標準） |
| 512×512 | 2x | 1024×1024 | システム設定（Retina） |

**推奨制作サイズ**: 1024×1024px（2xスケール）をベースに、各サイズにリサイズ

## 3. デザインコンセプト

### 3.1 基本方針

- **シンプルで認識しやすい**: メニューバーで小さく表示されても識別可能
- **集中・時間の概念を表現**: タイマーや時計の要素を含む
- **Apple Human Interface Guidelines準拠**: macOSのデザイン言語に合致
- **視認性重視**: メニューバーで常に表示されるため、目立ちすぎず、でも見つけやすい

### 3.2 デザインキーワード

- シンプル
- ミニマル
- 集中
- 時間
- ポモドーロ（オプション）

---

## 4. 視覚的要件

### 4.1 カラースキーム

- **推奨**: 
  - システムカラーを活用（ダークモード対応）
  - または、シンプルなモノクロ/アクセントカラー1色
- **避けるべき**: 
  - 多色使いすぎ
  - コントラストが低い配色

### 4.2 アイコン要素

**必須要素:**
- タイマー/時計の視覚的表現
  - 例: 円形のタイマー、時計の針、砂時計、ストップウォッチなど

**オプション要素:**
- 集中を表現する要素
  - 例: 集中している人のシルエット、フォーカスリングなど
- ポモドーロの要素（トマトなど）

### 4.3 スタイル

- **フラットデザイン**: 3D効果や影は最小限に
- **角丸**: macOSのアイコンは角丸が標準（自動適用される場合あり）
- **余白**: アイコン全体の10-15%程度の余白を確保

---

## 5. 状態別表示要件

メニューバーアイコンは、アプリの状態に応じて動的に変更されます：

| 状態 | 表示内容 | 備考 |
|------|---------|------|
| **非稼働時** | 静的なタイマーアイコン | デフォルト状態 |
| **稼働中** | 残り時間を「MM:SS」形式で表示 | アイコンと時間を併記、または時間のみ |
| **一時停止中** | 一時停止マーク（⏸）+ 残り時間 | 例：「⏸ 15:30」 |

**アイコンデザインへの影響:**
- アイコン自体は静的なデザインでOK（状態はテキストで表現）
- ただし、稼働中にアイコンをアニメーション化する場合は考慮が必要

---

## 6. 技術的要件

### 6.1 ファイル形式

- **形式**: PNG（推奨）または PDF（ベクター）
- **色空間**: sRGB
- **透明度**: アルファチャンネル対応（背景透明可）

### 6.2 ファイル配置

- **配置先**: `TimerApp/TimerApp/Assets.xcassets/AppIcon.appiconset/`
- **ファイル名**: `icon_16x16.png`, `icon_16x16@2x.png` など
- **設定ファイル**: `Contents.json` に各サイズのファイル名を指定

### 6.3 命名規則

推奨形式:
```
icon_16x16.png          (16×16, 1x)
icon_16x16@2x.png       (32×32, 2x)
icon_32x32.png          (32×32, 1x)
icon_32x32@2x.png       (64×64, 2x)
icon_128x128.png        (128×128, 1x)
icon_128x128@2x.png     (256×256, 2x)
icon_256x256.png        (256×256, 1x)
icon_256x256@2x.png     (512×512, 2x)
icon_512x512.png        (512×512, 1x)
icon_512x512@2x.png     (1024×1024, 2x)
```

## 7. デザイン案の方向性

### 案1: シンプルな円形タイマー
- 円形のタイマーリング
- シンプルな針やインジケーター
- ミニマルなデザイン

### 案2: ストップウォッチ
- クラシックなストップウォッチのシルエット
- シンプルな線画スタイル

### 案3: 砂時計
- 砂時計のアイコン
- 時間の経過を表現

### 案4: ポモドーロトマト
- ポモドーロ技法の象徴であるトマト
- シンプルなトマトのアイコン

### 案5: フォーカスリング
- 集中を表現するリング状のデザイン
- 中央に時間の要素を配置

## 8. チェックリスト

アイコン完成時に確認すべき項目：

- [ ] 全10サイズのアイコンが作成されている
- [ ] 各サイズで視認性が確保されている（特に16×16）
- [ ] ダークモードでも見やすい
- [ ] Contents.jsonに正しく設定されている
- [ ] ファイル名が命名規則に従っている
- [ ] メニューバーで実際に表示して確認
- [ ] Dock、Finder、システム設定でも表示確認
- [ ] Apple Human Interface Guidelinesに準拠している

## 10. 次のステップ

1. **デザイン案の選定**: 上記の案から選択、または新規デザイン
2. **デザイン制作**: 1024×1024pxでベースデザインを作成
3. **サイズ展開**: 各サイズにリサイズ・最適化
4. **実装**: Xcodeプロジェクトに配置
5. **テスト**: 実際のアプリで表示確認

---

## 補足

- アイコンは後から変更可能ですが、ユーザー体験に大きく影響するため、最初からしっかりと設計することが重要です
- メニューバーアプリの場合、16×16サイズでの視認性が特に重要です
- 必要に応じて、アイコンのデザインを複数案作成して比較検討することを推奨します
